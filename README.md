# MK-tool（中文版）

`MK-tool` 是一个基于 Chrome/Edge Manifest V3 的浏览器扩展，主要用于门店项目资料提取、三图资料下载、字段识别、默认文件管理，以及钉钉表单同步。

当前仓库版本对应扩展版本 `1.1.3`，可直接作为开发目录加载到浏览器，也可以打包为 ZIP 分发。

## 功能概览

- 一键提取页面中的图片和文件并下载
- 三图资料下载
- 自动提取页面字段信息并在悬浮面板中展示
- 同步到钉钉表单
- 一键打开钉钉表格/表单地址
- 配置默认文件并随下载目录一并管理
- 维护“序号 / 门店编码 / 上匠门店简称 / 月份”映射
- 支持导入、导出、清空门店映射表

## 适用环境

- Google Chrome
- Microsoft Edge
- 支持加载 Manifest V3 扩展的 Chromium 内核浏览器

## 快速开始

### 1. 加载扩展

1. 打开浏览器扩展管理页：
   - Chrome：`chrome://extensions`
   - Edge：`edge://extensions`
2. 开启“开发者模式”
3. 选择“加载已解压的扩展程序”
4. 选择当前项目目录 `MK_tools`

也可以直接运行：

```bat
install.bat
```

该脚本会把当前目录复制到本机 `AppData` 下，并自动打开扩展管理页，方便重新加载。

### 2. 打开设置页

加载完成后，打开扩展设置页，配置以下内容：

- 钉钉表单地址：用于“同步表单”
- 钉钉表格地址：用于“打开表单”
- 默认文件目录
- 默认文件列表
- 序号门店映射表

## 使用说明

### 页面悬浮面板

在目标页面中，扩展会注入悬浮面板，可用于：

- 提取页面字段
- 一键提取并下载
- 下载三图资料
- 同步钉钉表单
- 打开钉钉表单页面

默认快捷键：

```text
Alt + Q
```

用于打开或关闭页面悬浮面板。

### 一键提取并下载

适用于从当前页面提取图片、附件和相关资料，并按门店目录进行下载归档。

### 三图资料下载

用于从页面中的“效果图 / 施工图 / 三图相关区域”收集可下载文件，并落到对应目录。

### 字段提取

扩展会尝试从页面中提取以下常见字段：

- 门店编码
- 上匠门店简称
- 加盟商名称
- 加盟商联系方式
- 预估流水
- 外卖占比
- 外卖流水
- 完整地址
- 项目类型
- 省份
- 拓展顾问
- 接案时间
- 店宽
- 装修面积
- 完成时间
- 状态

### 钉钉表单同步

配置好设置页中的钉钉地址后，可以在悬浮面板中：

- 点击“同步表单”自动打开新表单并填充字段
- 点击“打开表单”直接跳转到配置好的钉钉表格/表单地址

相关设计说明可参考：

- [表单同步优化说明](./docs/%E8%A1%A8%E5%8D%95%E5%90%8C%E6%AD%A5%E4%BC%98%E5%8C%96%E8%AF%B4%E6%98%8E.md)

## 项目结构

```text
MK_tools/
├─ manifest.json            扩展清单
├─ background.js            后台 service worker
├─ content.js               页面内容脚本与悬浮面板逻辑
├─ content-data-field-id.js 字段补充逻辑
├─ popup.js                 弹出页逻辑
├─ index.html               扩展弹出页
├─ options.html             设置页入口
├─ options.js               设置页主逻辑
├─ options-form-url.js      钉钉地址配置逻辑
├─ assets/                  构建产物与页面资源
├─ docs/                    说明文档
├─ pack.bat                 Windows 打包入口
├─ pack.ps1                 ZIP 打包脚本
└─ install.bat              本地安装辅助脚本
```

## 打包发布

运行：

```bat
pack.bat
```

或者：

```powershell
powershell -ExecutionPolicy Bypass -File .\pack.ps1
```

打包脚本会读取 `manifest.json` 中的版本号，并在项目上一级目录生成：

```text
MK-tool-版本号.zip
```

## 数据存储说明

扩展主要通过 `chrome.storage.local` 保存以下内容：

- 默认文件
- 默认文件目录
- 钉钉表单地址
- 钉钉表格地址
- 门店编码与序号映射
- 门店编码与简称映射
- 门店编码与月份映射

## 开发说明

- 当前仓库已提交为可直接运行的扩展目录，不依赖额外的前端开发服务即可加载
- `assets/` 中包含设置页等相关构建资源
- 修改核心逻辑后，需在扩展管理页重新加载扩展，再刷新目标页面验证

## 常见操作

### 重新加载扩展

当你修改了以下文件后，通常需要重新加载扩展：

- `manifest.json`
- `background.js`
- `content.js`
- `options.html`
- `options.js`
- `options-form-url.js`

操作方式：

1. 打开扩展管理页
2. 找到 `MK-tool`
3. 点击“重新加载”
4. 回到目标页面刷新网页

### 更新版本号

扩展版本号位于：

```json
manifest.json
```

修改 `version` 后，再重新打包即可。
