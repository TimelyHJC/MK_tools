# 表单同步逻辑优化说明

## 一、MD 文档与现有逻辑对比

### 文档《已创建了浏览器插件，获取信息。需要能填入修改表单.md》要点

| 建议 | 当前 MK 插件 |
|------|-------------|
| **① data-field-id 优先** | ✅ 已支持常见 `data-field-id` 优先匹配（内置字段别名） |
| **② input + change 事件** | ✅ 已用 `nt()` 触发 input 与 change |
| **③ contenteditable 用 innerHTML + input** | ✅ 已支持 |
| **④ MutationObserver 防异步** | ❌ 未做，仅延迟重试 |
| **⑤ 防重复填入锁** | ❌ 未做 |

### 结论

**当前以保留“同步表单（新开表单并填入）”为主**。在保留现有逻辑的前提下，可继续增加：

1. **data-field-id 优先策略**：表单有 `data-field-id` 时优先用其定位
2. **字段映射配置**：暂不提供设置页中的「插件字段名 → data-field-id」自定义映射，当前仅保留内置字段别名
3. **添加门店同步**：通过「同步表单」打开新表单并填入字段

## 二、功能设计

### 1. 添加门店信息（沿用现有逻辑）

- 流程：提取字段 → 点击「同步表单」→ 打开新表单页 → 填入 → 提交
- 当前已加入常见 `data-field-id` 优先定位，未命中时仍回退到现有标签/placeholder 逻辑

## 三、钉钉字段名与 data-field-id 内置映射（示例）

当前插件仅保留内置字段别名；如需排查实际表单字段，可在控制台执行：

```javascript
[...document.querySelectorAll('[data-field-id]')].map(el => el.getAttribute('data-field-id')).filter(Boolean)
```

内置常见映射示例：

| 插件字段 | 常见 data-field-id |
|---------|---------------------|
| 链接 | link / page_url |
| 门店编码 | store_code / store_id |
| 上匠门店简称 | store_short_name |
| 加盟商名称 | franchisee_name |
| 加盟商联系方式 | contact_phone |
| 预估流水 | est_revenue |
| 外卖占比 | takeout_ratio |
| 外卖流水 | takeout_revenue |
| 完整地址 | full_address |
| 项目类型 | project_type |
| 省份 | province |
| 拓展顾问 | consultant |
| 接案时间 | receive_time |
| 店宽 | store_width |
| 装修面积 | decorate_area |
| 完成时间 | complete_time |
